Huidige Octopress Setup

Oct 2nd, 2013

De vorige publicatie ging niet zoals ik dacht. Het bleek dat de huidige Octopress setup opgezet was om te publiceren op mijn Github blog. Derhalve moeten de volgende stappen gevolgd worden:

post "Huidige Octopress Setup" --dest octopress
cd ~/octopress
rake generate
rake deploy

De url blog.hahoweb.nl wijst naar mijn Github account whofman.github.com via een CNAME bij mijn domain provider.

Stappen Voor Nieuwe iMac

Oct 2nd, 2013

Ik heb een nieuwe iMac besteld met een Fusion drive. Ik wil eigenlijk hierop een schone installatie doen. Maar een aantal applicaties wil ik gewoon overzetten, zoals Octopress en mijn pritlog/blog systeem. Aangezien het een half jaar geleden was dat ik voor het laatst iets met beide systemen deed, ben ik het overzicht over de diverse folders enigszins kwijt geraakt. Dat moet ik nu oppoetsen. Hiertoe begin ik met het (proberen te) posten van dit bericht:

post "Stappen voor een nieuwe iMac" --dest octopress
cd ~/octopress
rake generate
cp -r public/* /Volumes/Web/octopress

Dit waren de handmatige stappen om tot publicatie te komen voor zover ik me herinner. Ben benieuwd!

Octopress Hernieuwd

May 9th, 2013

Na een lange onderbreking heb ik weer eens naar Octopress gekeken. Op de website was te zien dat de ontwikkelaars niets gedaan hadden. Dus, dacht ik, kon ik de installatie files gemakkelijk naar mijn OS X Webserver overzetten. Helaas was dat te gemakkelijk gedacht. De Ruby versie was van 1.8.7 naar 2.1.1 gewijzigd. Ook was iconv in deze versie deprecated. Na wat zoeken bleek dat er niet ee-twee-drie een oplossing voor was. Ik heb dus op goed geluk de aanroep van iconv uit maraku/input/parse_doc.rb gesloopt. Ik zal wel zien wanneer en hoe dit fout gaat.

Mijn tests lieten zien, dat een en ander nog steeds werkte. Het staat me nu dus vrij om weer met Octopress aan de gang te gaan.

Enkele Laatste Wijzigingen

Dec 28th, 2012

Ik heb nog enkele wijzigingen in Octopress aangebracht, voornamenlijk om de teksten te “vernederlandsen”. Deze betreffen de modules:

_config.yml
source/_includes/navigation.html
source/_includes/custom/navigation.html
source/_includes/custom/asides/about.html
source/_includes/custom/asides/category_list.html

Ook heb ik een bash opdracht deploy (~/bin/deploy) aangemaakt om de gegenereerde files naar mijn lokale (Mac) server te synchroniseren.

Overigens, rake automatisch laten verlopen is moeilijker dan gedacht.

Tenslotte, de Octopress manier om “posts” aante maken via rake new_post “Titel” vond ik zeer onhandig. Daarom heb ik een andere opdracht post (~/bin/post) gemaakt, die niet alleen de correcte header file creëert, maar dit tevens doet in de juist directory:

post "titel" [--tag tags, …] [--cat cats, ...]

Octopress Bijna Klaar

Dec 10th, 2012

Na het toevoegen van Tags is mijn inplementatie van Octopress bijna klaar. Ik wil nog een Lees meer optie hebben om te lange posts te vermijden op de main pagina. Dan heb ik gemerkt dat de Tags niet getoond worden op de Archief pagina, dus dat wil ik ook nog aanpassen.

Het splitsen van lange posts, het eerste gedeelte op de main pagina, de rest als volledig artikel kan je bereiken met de more tag: <!–more–>

Het toevoegen van posts gebeurt nu met de commandline utility post. Echter, daarna moet ik nog handmatig (in ~/octopress) de commando’s rake generate en rake deploy uitvoeren. Dat wil ik ook nog automatiseren via Hazel en/of Launchd/Lingon.
